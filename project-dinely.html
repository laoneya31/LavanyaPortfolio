Dinely

Process

Dinely is a mobile app that helps users discover nearby restaurants, browse menus, and reserve tables all in one place. The app personalizes recommendations based on dietary restrictions and cuisine preferences, combining real-time restaurant data with a seamless reservation experience. It is built with JavaScript and React, and tested on Expo Go.

I designed and implemented the end-to-end authentication flow using Firebase Authentication. This included sign-in, sign-up, password reset, and a four-step onboarding process that collected the user's name, dietary restrictions, cuisine preferences, and login credentials. Furthermore, I set up and managed the cloud database layer responsible for persisting user profiles beyond a single device session. Dietary and cuisine preferences, profile edits, and account details are all synced to Firestore in real time, giving users a consistent experience across logins.

Additionally, lightweight on-device data persistence using Async Storage for UI preferences like color theme, appearance mode (light/dark/system), and notification settings was also added. Finally, I integrated native camera access to allow users to take and upload a profile photo directly from their device. Handled permission requests, image capture, and local storage of the selected image so it persists in the user's profile view.

Working collaboratively on GitHub for the first time introduced friction early on due to frequent merge conflicts caused by team members committing directly to main. We resolved this by adopting a branch-based workflow, which dramatically reduced conflicts and improved our parallel development process. We also struggled with integrating local and cloud storage, as some properties, like reservations, that were stored locally, would show up across accounts on the same device. We linked all local storage to the cloud using the unique ID, which removed this issue.

The final app delivered a fully functional, personalized dining discovery experience with real authentication, cloud-synced profiles, local data persistence, and native camera support, all with a clean, polished, and easy-to-navigate design.
