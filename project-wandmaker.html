Wandmaker's Atelier

Process

The aim of this project was to build an interactive wand-crafting simulation in Java, where users make sequential material and design choices through a mouse-driven GUI, with each decision dynamically shaping the wand's final power and appearance. The brief required blending solid software design patterns with generative visuals to create an experience that felt immersive and polished rather than purely functional.

I built the full simulation independently, implementing the Decorator design pattern to layer each user choice so that material and customization decisions accumulated meaningfully into a calculated final outcome. To bring the world to life, I generated complex animated visuals using recursion and Perlin noise, producing fractal patterns, branching effects, and organic particle movement. I also programmed a day and night cycle with smooth transitions and designed four distinct spell effects, each paired with synthesized audio feedback. I also conducted iterative peer testing throughout development, using that feedback to refine hover states and contextual tooltips so users were guided naturally through each stage.

One of the main challenges was implementing the Decorator pattern correctly. Getting each material and customization choice to layer onto the wand in the right order, while still calculating an accurate final outcome, took several rounds of restructuring before the logic held up across all the different choice combinations.

The other major challenge was the Perlin noise generation for the wand's visual effects. Getting the noise to produce organic, on-theme patterns instead of random or flat-looking results took a lot of trial and error. I spent significant time adjusting parameters, testing outputs, and watching tutorials on YouTube to understand how the underlying algorithm worked before I could tune it to actually look like it belonged in the world I was building.

The result is a fully playable, visually polished interactive simulation that demonstrates both software architecture knowledge and a strong sense of atmospheric design, with the technical and creative decisions reinforcing each other throughout. This project reflects my ability to apply structured programming patterns to generative, visually driven systems, and to treat technical implementation and aesthetic intent as part of the same problem rather than separate concerns.
